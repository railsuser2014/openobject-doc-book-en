.. i18n: .. describe:: Preventing SQL Injection:

.. describe:: Preventing SQL Injection:

.. i18n: Care must be taken not to introduce SQL injection vulnerabilities to SQL
.. i18n: queries. SQL Injection is a kind of vulnerability in which the user is able to
.. i18n: introduce undesirable clauses to a SQL query (such as circumventing filters or
.. i18n: executing **UPDATE** or **DELETE** commands) due to incorrect quoting in
.. i18n: the application.

Care must be taken not to introduce SQL injection vulnerabilities to SQL
queries. SQL Injection is a kind of vulnerability in which the user is able to
introduce undesirable clauses to a SQL query (such as circumventing filters or
executing **UPDATE** or **DELETE** commands) due to incorrect quoting in
the application.

.. i18n: In order to prevent SQL injection you need to be cautious when constructing SQL
.. i18n: query strings. Good advices are to use %d and %f when only numbers are to be
.. i18n: substituted and always use psycopg formatting parameters. For example the
.. i18n: following expression is incorrect:

In order to prevent SQL injection you need to be cautious when constructing SQL

.. i18n: Here are some rules to respect:

Here are some rules to respect:

.. i18n: * many2one fields should respect this regex: '.*_id'
.. i18n: * one2many fields should respect this regex: '.*_ids'
.. i18n: * one2many relation table should respect this regex: '.*_rel'
.. i18n: * many2many fields should respect this regex: '.*_ids'
.. i18n: * use underscore to separate words
.. i18n: * avoid using uppercases
